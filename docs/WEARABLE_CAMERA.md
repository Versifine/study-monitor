# 全天可穿戴摄像头路线

> Version 1 边界：可穿戴硬件、固件和专用接入均不实现；正式备考前也不要求完成最终硬件。

## 1. 目标

希望接近手表级佩戴感，并尽量全天佩戴。

## 2. 现实约束

无法同时获得：

- 手表级重量
- 单模块 24 小时续航
- 连续高清录像
- 足以看清纸面小字

因此可穿戴设备不承担精细纸面识别。

## 3. 推荐架构

```text
胸前轻量传感头
  ├─ 低频摄像头
  ├─ 麦克风
  ├─ IMU
  ├─ 环形缓存
  └─ 事件触发
        ↓ BLE / Wi-Fi
手机或腰间计算模块
        ↓
本地服务器 / 云端分析
```

## 4. 场景接管

- 全天普通场景：低频哨兵
- 检测到学习：书桌摄像头接管纸面
- 检测到电脑操作：屏幕和活动采集器接管
- 检测到运动或进食：可穿戴设备保留上下文

## 5. 原型顺序

1. 旧手机或开发板验证数据价值
2. Pi Zero 类设备验证分段录像与上传
3. 小型传感头验证佩戴
4. 只有在学习闭环有效后再做正式硬件

正式备考前不要求完成最终硬件。

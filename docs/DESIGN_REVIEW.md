本次完整审查 `AGENTS.md`、`MASTER_SPEC.md` 列出的全部文档、M0-M6 任务提示和仓库入口文档，只修改设计文档，不实现业务代码，也不进入 M0。

## 3. 补齐的降级路径

- 覆盖率投影失败：保留权威事实，标记陈旧，稍后重建
- 数据库不可写：不确认新写入，保留来源/临时文件，停止删除
- 磁盘预警/关键/保留空间威胁：逐级停止可选工作、媒体接收和确认写入
- 备份失败：不替换最近有效备份；超过 RPO 才升级 P1
- 回滚 schema 不兼容：安全拒绝并进入 Minimum mode，不执行 down migration
- 仪表盘失败：写入 API、后台任务和运维脚本继续
- 外部云端/分析失败：显示关闭或积压，Recorder Core 不受影响

完整矩阵见 `FAILURE_POLICY.md`。

## 4. 删除或拒绝的不必要复杂度

- V1 不创建分析 worker、模型队列、Python 环境、Observation/Inference/知识图谱业务表
- V1 不开发自定义录像器、原生手机端、健康平台连接器或远程公网服务
- V1 不引入微服务、消息中间件、容器编排或独立前端服务
- 仪表盘从采集器心跳读取外部积压；分析关闭时显示状态，不为“待分析”提前建队列
- 建议目录不预建 `analysis/`、`models/` 等未来目录

## 6. 仍需在实现阶段落值、但不再属于产品开放问题

以下值依赖目标 Windows 机器和外部采集器，允许在 M4 使用经过测试的安全默认值，在 M6 开始前写入冻结清单：

- 心跳周期、允许迟到、离线判定和计划启用时段
- 磁盘预警/关键/数据库保留空间
- 最短媒体保留期和所需备份状态
- RPO、恢复/自动恢复 RTO 和资源上限

这些字段和关系不能省略，M6 期间不能改变。长期归档时长、硬件、模型、题库和主动追问形式仍属于 `OPEN_QUESTIONS.md` 的后续研究。

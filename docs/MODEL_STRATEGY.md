# 本地与云端模型分工

> Version 1 边界：不安装或运行模型、Python 分析进程与模型任务队列。record-only 和 minimum 不依赖 RTX 4060、云端或本文件中的模型能力。

## 1. 本地 RTX 4060

适合：

- 场景分类
- 人体、纸张、手部检测
- 画面变化检测
- 相似帧去重
- 视频切片
- 嵌入和轻量视觉理解
- 云端失败时降级

不适合逐帧理解全天高分辨率视频。

## 2. 云端视觉模型

负责：

- 多图和短视频理解
- 手写过程语义
- 题目与步骤识别
- 环境和行为判断

必须输出结构化结果和置信度。

## 3. 文本推理模型

低成本文本模型可负责：

- 独立解题
- 题目审查
- 知识点标注
- 变式生成
- 多模型交叉验证
- 周报与计划解释

视觉输入不能依赖不支持视觉的模型。

## 4. 模型版本管理

每个结果保存：

- 模型名称
- 模型版本
- 提示词版本
- 输入证据引用
- 输出 schema 版本
- 置信度

## 5. 失败降级

- 云端失败：保留队列，不通知普通积压
- 视觉失败：保存关键帧，标记待重算
- 推理冲突：降低置信度或拒绝判断
- 成本过高：只分析测评和关键学习片段

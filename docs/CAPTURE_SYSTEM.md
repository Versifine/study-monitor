# 采集系统设计

> Version 1 边界：只接入 ActivityWatch、通用 JSON Evidence 和外部已完成媒体的受管导入。纸面跟踪、摄像头自检、手机原生采集、可穿戴和内容分析均属后续；V1 只可保存外部采集器上报的确定性质量状态。

## 1. 数据源

M3 已实现的 Recorder Core 接入边界只有：ActivityWatch 指定 bucket 的 GET-only 导入、版本化通用 JSON Evidence、版本化心跳和 M2 已完成媒体。以下条目描述允许保存的外部 Evidence；不代表 Recorder Core 在 M3 内实现对应原生采集器或分析。

### 电脑

- 活动窗口
- 浏览器标签和 URL
- PDF 和课程页面
- IDE 项目与文件
- 键鼠活跃/离开状态
- 可选屏幕录制

不需要保存密码或原始按键内容。

### 书桌

- 固定俯拍视频
- 高频关键帧或高分辨率照片
- 手和纸张跟踪
- 翻页、擦改和离桌事件

### 手机

- 前台应用
- 解锁
- 使用时间段
- 健康数据同步

Version 1 只接受外部工具经 `/api/v1/evidence/batch` 转换后的上述事实，不开发手机端或健康平台连接器。

### 未来可穿戴设备

- 全天环境上下文
- IMU
- 麦克风
- 低频视觉哨兵
- 事件触发高质量片段

## 2. 媒体策略

- 连续记录可以分段保存
- 分析按事件触发
- 先检测场景变化和纸面变化
- 强模型只处理关键帧和短视频
- 原始媒体与分析结果分离

## 3. 数据覆盖率

必须区分：

- 确认无活动
- 设备离线
- 相机失焦
- 画面遮挡
- 文件损坏
- 上传延迟
- 分析积压

M3 实际投影把可用性冻结为 `covered`、`confirmed_idle`、`pending`、`delayed`、`offline`、`unknown` 单值，把失焦/遮挡、损坏、时钟不确定和不完整作为独立质量标记。Version 1 只消费外部采集器的确定性质量上报；无心跳/AFK 事实不得推断为空闲。实现合同见 [`COLLECTORS_AND_TIMELINE.md`](COLLECTORS_AND_TIMELINE.md)。

## 4. 摄像头自检

以下均为后续外部采集器或研究能力，M3 不实现视觉检测：

- 清晰度
- 对焦
- 曝光
- 运动模糊
- 纸张占比
- 遮挡比例
- 镜头污损
- 书写区域出框

## 5. 全量记录与分析成本

完整录像可以保存，但不逐帧调用云端模型。使用：

- 感知哈希去重
- 场景切换检测
- 关键帧
- 纸面差分
- 重要时段升采样

# 采集系统设计

> Version 1 边界：只接入 ActivityWatch、通用 JSON Evidence 和外部已完成媒体的受管导入。纸面跟踪、摄像头自检、手机原生采集、可穿戴和内容分析均属后续；V1 只可保存外部采集器上报的确定性质量状态。

## 1. 数据源

### 电脑

- 活动窗口
- 浏览器标签和 URL
- PDF 和课程页面
- IDE 项目与文件
- 键鼠活跃/离开状态
- 可选屏幕录制

不需要保存密码或原始按键内容。

### 书桌

- 固定俯拍视频
- 高频关键帧或高分辨率照片
- 手和纸张跟踪
- 翻页、擦改和离桌事件

### 手机

- 前台应用
- 解锁
- 使用时间段
- 健康数据同步

### 未来可穿戴设备

- 全天环境上下文
- IMU
- 麦克风
- 低频视觉哨兵
- 事件触发高质量片段

## 2. 媒体策略

- 连续记录可以分段保存
- 分析按事件触发
- 先检测场景变化和纸面变化
- 强模型只处理关键帧和短视频
- 原始媒体与分析结果分离

## 3. 数据覆盖率

必须区分：

- 确认无活动
- 设备离线
- 相机失焦
- 画面遮挡
- 文件损坏
- 上传延迟
- 分析积压

## 4. 摄像头自检

- 清晰度
- 对焦
- 曝光
- 运动模糊
- 纸张占比
- 遮挡比例
- 镜头污损
- 书写区域出框

## 5. 全量记录与分析成本

完整录像可以保存，但不逐帧调用云端模型。使用：

- 感知哈希去重
- 场景切换检测
- 关键帧
- 纸面差分
- 重要时段升采样

# 项目背景与已确认需求

## 1. 用户目标

- 主目标：服务于 2028 年考研准备
- 计划在 2026 年 9 月开始持续备考
- 2026 年 9 月不是 Recorder Core 的强制上线截止日；在 M6 通过前继续使用现有工具，不为赶日期压缩验收
- 希望系统尽量全自动，不要求每天打卡、写日志或手动填写状态
- 希望记录学习、生活行为、学习进度、心理状态、健康状态和饮食习惯
- 接受完整记录，不把个人隐私作为首要限制
- 允许使用云端模型，本地 RTX 4060 作为算力补充
- 不愿将纸笔学习迁移到平板或数位板
- 允许系统主动安排题目、追问和延迟复测
- 希望可穿戴记录设备接近手表级佩戴体验，并尽可能全天佩戴

## 2. 最重要的现实约束

进入正式备考后，用户不能继续开发软件本体。系统出现普通 Bug、分析效果不好或功能不完善时，不能诱导用户打开 Codex 继续调试。

因此系统在成为正式备考基础设施前必须：

- 在正式备考前冻结
- 分离稳定采集与实验分析
- 允许分析层长期停用
- 失败后自动降级
- 可回滚到稳定版本
- 用固定维护窗口代替即时调试

## 3. 核心判断

### 3.1 被动观察不能直接证明掌握

摄像头、屏幕、停留时间和心率可以记录行为，但无法直接读取脑内理解。

系统必须把学习状态拆成：

- 接触了什么
- 实际练习了什么
- 是否独立完成
- 当前正确表现
- 延迟后是否保持
- 陌生变式能否迁移

### 3.2 摄像头角色分工

- 胸前摄像头：补全全天环境和行为上下文
- 书桌固定俯拍摄像头：记录题目、纸面、笔迹变化、擦改和翻页
- 电脑采集器：记录软件、网页、PDF、AI 工具和答案来源
- 手机采集器：记录前台应用、解锁、搜索和通知上下文

胸前摄像头不承担精细手写内容识别。

### 3.3 视觉模型优先于 OCR

纸面主链路不应是“先 OCR，再让语言模型理解”。

正确路线是视觉模型直接观察：

- 题目区域
- 书写前后状态
- 新增和划除区域
- 短时间片段
- 是否翻书、看答案或使用其他信息源

OCR 只用于辅助检索和索引。

### 3.4 题目质量必须分级

- A 级：考研真题、官方样题
- B 级：可靠教材和主流习题集
- C 级：基于 A/B 题受约束生成并验证的变式题
- D 级：AI 自由生成题，只能低权重试用

掌握度不得由未经验证的 AI 题单独决定。

## 4. 已确认的产品定位

产品不是“实时评价人生的总分系统”，而是：

> 稳定保存学习和生活证据；用经过验证的作答、复测和题库证据评估考研进度；复杂分析失败时不影响学习和记录。

## 5. 必须接受的不完美

- 可穿戴设备可能漏录
- 视觉模型会误判
- 纸面数学无法保证逐字符正确识别
- 24 小时、手表级重量、连续高清录像和足够清楚的小字记录无法同时达到
- 心理状态只能做行为偏离预警，不能作为医学诊断
- 饮食热量只能估算区间

这些不足不构成备考期间必须调试的理由。

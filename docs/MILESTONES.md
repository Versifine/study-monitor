# 开发里程碑

## 通用完成门槛

每个里程碑必须同时满足：本节验收项、确定性单元测试、相关集成测试、`scripts/test.ps1`、`scripts/smoke.ps1`、相关文档更新、无关改动为零、diff 已审查、剩余风险已列出，并可安全创建 Git 检查点。未满足时不得开始下一里程碑。

## M0：仓库与开发工具

### 范围

Go 模块、单一可执行程序骨架、配置、结构化日志、版本信息、Windows 原生开发脚本和测试骨架。

### 明确排除

SQLite 业务 schema、事件写入、媒体登记、采集器适配、前端和 AI。

### 验收标准

- `go.mod`、`go.sum` 和工具版本固定；构建不依赖 Docker、WSL 或云服务
- 只支持版本化 JSON 配置和环境变量覆盖；有安全默认值、示例文件、启动前校验，运行数据默认写入 Git 忽略目录
- 默认只监听 loopback；非 loopback 必须显式配置且不作为 Version 1 验收路径
- 结构化日志至少包含 UTC 时间、级别、组件、事件名、构建版本和稳定错误码；不得记录 Evidence 内容或密钥
- `--version` 输出版本、提交和构建时间；`--check-config` 只校验且不写运行数据
- Windows PowerShell 下 `scripts/dev.ps1`、`test.ps1`、`build.ps1`、`smoke.ps1` 均可从干净工作树非交互运行
- M0 smoke 只验证配置、启动、进程存活和干净退出；可以有不依赖数据库的 `/health/live`，不得伪装 M1 的存储就绪检查
- 脚本在临时目录运行，失败时返回非零，成功或失败后均不遗留后台进程
- `scripts/test.ps1` 覆盖配置边界、日志字段和版本输出；`scripts/build.ps1` 产出可在 Windows 原生启动的单一核心二进制

## M1：仅追加事件存储与本地 API

### 范围

SQLite WAL、只前向迁移、仅追加事件、幂等批量接收、本地查询、存储就绪健康检查和确定性分页。

### 明确排除

媒体、ActivityWatch 适配、覆盖率、AI、分析 schema、掌握度和复杂认证。

### 验收标准

- 新数据库和已存在数据库均能幂等迁移；Version 1 迁移只做与上一稳定版兼容的新增，不 drop/rename 旧结构或改变字段语义；启用 WAL、外键和有界 busy timeout
- 事件原样保存设备时间表示，同时保存无歧义的设备 UTC、接收 UTC、时钟偏移与误差、collector、schema 版本和 payload hash；缺少必要时区信息的输入被拒绝
- `(collector_id, idempotency_key)` 唯一：同键同内容返回原事实，同键不同内容返回冲突，均不覆盖原数据
- 结构合法的批次返回与输入顺序对应的 `accepted`、`duplicate`、`rejected` 或 `conflict` 逐项结果；一个损坏项不阻塞其他合法项
- 请求体、批量条数、单事件、payload 深度、并发和查询页大小均有配置上限，超限可预测地拒绝
- 写入端点只接受明确版本的 JSON；默认 loopback 且不允许任意浏览器 Origin，防止本地网页静默提交
- 写入确认只在事务提交后返回；服务被终止并重启后，已确认事实仍存在，未确认请求可安全重试
- 查询使用稳定排序和不受后续插入扰动的游标；相同快照重复翻页不漏项、不重复
- `/health/ready` 区分进程存活、数据库可写、迁移失败和降级只读状态
- 集成测试覆盖重复提交、幂等冲突、部分损坏批次、并发同键、写入中止、重启、数据库忙和迁移失败
- smoke 在临时数据库完成“写入、重复重放、查询、重启再查询”，不接触真实数据目录

## M2：媒体分段登记与完整性

### 范围

外部媒体受管入口、版本化 sidecar、就绪标记、目标卷暂存、校验、隔离、幂等确认和中断恢复。

### 明确排除

自制摄像头/屏幕录像器、任意外部路径长期引用、视频内容分析和破坏性保留执行。

### 验收标准

- 发布并验证 sidecar schema，至少包含来源幂等键、collector、带时区/偏移的设备起止时间及原始表示、大小、SHA-256、媒体类型、schema 版本和完成标记
- Version 1 认证使用不超过 10 分钟的媒体段；超出配置上限的段可见地拒绝或隔离，不静默接受
- 没有就绪标记、仍在增长、超限、sidecar 缺失或不匹配的文件不得进入 `accepted`
- 入口/目标根目录规范化，目的路径由 Core 生成；拒绝路径穿越以及指向根目录外的符号链接、junction/reparse point
- 使用固定版本 `ffprobe` 验证时长、媒体类型和编码；工具缺失只停用媒体入口，不停止事件接收
- 内容先复制到目标卷临时文件，落盘并校验后在同卷原子改名，随后数据库提交 `accepted`；跨卷不假设原子移动
- 数据库不得出现指向不存在文件的 `accepted`；文件已落盘但数据库未提交时可在重启后收敛
- 同一来源键和相同校验和幂等返回同一媒体；同键不同内容或同内容冲突元数据可见地失败
- 损坏和不合规媒体进入受管隔离并保存原因；未知和未确认来源文件不被删除
- Recorder Core 只写确认标记，来源文件默认由来源采集器确认后清理
- 媒体导入事件和已接受媒体状态只追加；入口积压与媒体当前状态投影均可从历史重建
- 测试夹具覆盖有效、重复、截断、伪造大小/校验和、超限、路径穿越/reparse point、写入中断、目标改名后数据库失败、重启恢复和隔离失败
- M2 smoke 使用小型固定媒体完成导入、重复重放、隔离和重启恢复

## M3：采集器适配与统一时间线

### 范围

ActivityWatch 只读适配、通用 JSON Evidence 适配、追加心跳、时钟元数据、覆盖率投影和统一时间线查询。

### 明确排除

键盘记录器、密码或原始按键、原生手机/健康连接器、知识图谱和任何分析判断。

### 验收标准

- ActivityWatch 导入不修改其数据库；断点水位持久化，重启和重扫不产生重复事实
- 通用 JSON 使用版本化 schema 和与 M1 相同的幂等、大小及时间合同；手机/健康数据只能走此入口
- 心跳是追加事实；每个启用采集器配置心跳周期、允许迟到、离线时间和计划启用时段，关系校验通过
- 核心来源 SLA 校验执行 V1 下限：ActivityWatch 单次非计划离线/未知不超过 5 分钟；媒体段不超过 10 分钟且单次非计划无有效媒体不超过 15 分钟
- 心跳重复提交、媒体/故障状态重放和投影重建均遵守幂等合同，不产生重复事实或重叠区间
- 覆盖率按采集器生成不重叠的 `[start, end)` 区间；可用性和质量标记遵循 `V1_FREEZE_SPEC.md`
- 没有心跳时不得生成 `confirmed_idle`；迟到数据可把 `pending/delayed/unknown` 投影重算为最终状态且不改原始事实
- 时间线返回原始设备时间、设备 UTC、接收 UTC、校正时间及误差；时钟不确定时显式标记，并用稳定键确定顺序和分页
- 一个采集器离线、数据延迟、投影重建或适配器停用不影响其他来源写入和查询
- Record-only 和 Minimum mode 均有配置测试；Minimum mode 只启用 ActivityWatch、书桌媒体、核心健康和备份接口
- 固定测试数据覆盖空闲与离线区分、边界相接、重叠来源、迟到数据、时钟前后跳、夏令时显示和大误差
- smoke 生成多来源时间线，模拟一个来源离线，并验证覆盖缺口被准确暴露

## M4：恢复、保留、备份和回滚

**实现状态（2026-07-22）：已完成，等待 M6 使用冻结候选再次做 14 天认证。** M4 的实现、故障注入和 smoke 已覆盖本节全部验收项；本状态不表示 M5 或 M6 已开始。

### 范围

Windows 进程托管、模块隔离、有界重试、磁盘保护、保留执行、备份、恢复、安装卸载和上一稳定应用版本回滚。

### 明确排除

自动升级、破坏性 down migration、云端必需备份和无人确认删除未知数据。

### 验收标准

- `scripts/install.ps1` 和 `uninstall.ps1` 为当前用户注册/移除 Windows Task Scheduler 任务；登录后启动、失败后有界重启，默认不要求管理员权限，卸载不删除数据
- 非正常退出按有界退避重启；重复崩溃进入明确降级状态，记录故障且不形成无上限重启循环
- 可选模块能单独关闭并记录原因；覆盖率或仪表盘故障不停止事件与媒体接收
- 进程内适配器和媒体入口在暂时失败时保留未确认数据，恢复后按幂等合同继续
- 日志按大小/份数轮转，WAL checkpoint、入口积压和临时文件有上限或收敛策略；清理只作用于可证明无用的 Core 自有文件
- 磁盘预警、关键和数据库保留空间有安全默认值与关系校验；通过可注入磁盘状态测试，不要求真实填满磁盘
- 未启用自动保留时只降级、不删除；启用后也只能删除受管、已接受、超过最短保留期且满足备份条件的媒体，并追加删除状态
- `scripts/backup.ps1` 支持可验证的元数据备份和完整 Evidence 备份；产物包含一致数据库快照、配置、版本、媒体清单、校验和及明确的媒体覆盖范围
- `scripts/restore.ps1` 默认恢复到新目录，先验证再切换；原目录非显式确认不得覆盖
- `scripts/rollback.ps1` 只切换到上一稳定二进制和兼容配置，不回退或删改 Evidence schema；schema 不兼容时安全拒绝
- 每次发布保存构建与 schema 兼容清单，至少保证当前候选与上一稳定版本的回滚窗口
- 故障注入覆盖进程终止、数据库忙/不可写、网络断开、低磁盘、备份中断、恢复中断、坏备份和不兼容回滚
- smoke 分别验证自动恢复、Minimum mode、备份到新目录恢复并校验、以及上一稳定版回滚

## M5：最小只读仪表盘

**实现状态（2026-07-23）：已完成，等待 M6 使用冻结候选再次做 14 天认证。** M5 的离线构建、状态/API 契约、故障隔离、目标浏览器响应式验收和 smoke 已覆盖本节全部验收项；本状态不表示 M6 已开始计时。

### 范围

构建后嵌入 Go 二进制的静态 TypeScript 前端，只读展示时间线、覆盖率、采集器、存储、积压和故障。

### 明确排除

写入学习计划、实时 AI、社交功能、复杂设计系统、复杂动画和独立前端服务。

### 验收标准

- 前端依赖和构建工具版本固定，离线构建；生产运行不需要 Node.js
- 所有业务请求为只读查询；页面不提供修改 Evidence、保留策略、计划或模型状态的入口
- 明确区分 `disabled`、`not_installed`、`offline`、`delayed`、`unknown`、无数据和零积压，不把缺失渲染成正常
- 显示设备时间校正不确定性、覆盖质量标记、当前降级模式和数据最后更新时间
- 分析关闭时显示 `disabled` 或 `not_installed`，不创建分析队列，也不把它显示为“0 条待分析”
- 大时间线采用有界分页；慢查询、页面刷新和浏览器关闭不阻塞采集写入
- 仪表盘资源不存在或处理器关闭时，写入 API、健康检查和后台恢复继续工作
- 键盘可访问、颜色不是状态的唯一表达方式，在目标 Windows 浏览器上无需开发服务器即可使用
- 前端单元测试覆盖状态映射，API 契约测试覆盖空/错/迟到数据，smoke 打开嵌入页面并验证六类核心信息

## M6：十四天稳定性认证

**实现状态（2026-07-23）：认证工具已实现，14 天计时尚未开始。** 冻结候选、生产安装、ActivityWatch 和书桌媒体核心来源全部预检通过后，才允许生成密封清单并从下一个本地自然日开始；本状态不表示 M6 已完成。

### 范围

稳定性认证工具、非生产数据上的故障注入、冻结清单和最终报告。只修复冻结阻塞问题，不增加功能。

### 重新计时规则

任何二进制、依赖、核心配置、schema、保留行为或采集 SLA 的变更都使 14 天从第 1 天重新开始。只修改报告文字且不改变运行行为可以不重新计时，但必须记录。

### 验收标准

- 开始前固定候选版本、依赖、配置校验和、启用模块、采集 SLA、磁盘水位、资源上限、RPO/RTO 和上一稳定版本
- 同一候选构建和冻结清单连续运行 14 天，不增加功能、不放宽阈值
- 已确认事件和已接受媒体的 Recorder Core 数据丢失为 0，重复 Evidence 为 0，幂等冲突全部可见
- 每日及结束时数据库完整性检查通过；未完成或损坏媒体进入正确状态，`accepted` 媒体抽样校验通过
- 所有启用核心来源的计划时间在结算后 100% 被分类；外部离线和漏录作为覆盖缺口报告，不伪装成空闲
- 排除预先列明且另行验证 RTO 的故障注入窗口后，ActivityWatch 与书桌媒体的可用覆盖均不低于计划启用时间的 99%，并满足单次缺口上限；计划窗口不得为空
- 每种规定故障均在冻结 RTO 内自动恢复或进入规定降级；计划注入之外的人工代码修复次数为 0
- 磁盘增长、CPU、内存、句柄/协程、WAL、队列、临时文件和日志有界，没有随运行时间持续无界增长
- Record-only 贯穿认证；Minimum mode、所有可选模块单独关闭、备份恢复和上一稳定版回滚至少各演练一次
- 通知只在 P0/P1 门槛触发；P2/P3 不要求即时处理
- 故障注入至少覆盖网络断开、核心进程终止、系统重启、文件写入中断、重复提交、损坏媒体、低磁盘、可选云端不可用和时钟偏移
- 最终报告列出每日指标、故障与恢复、人工干预、覆盖缺口、恢复/回滚证据、遗留风险和最终通过/不通过结论

M6 通过后停止。不得自动进入纸面视觉理解、主动测评或其他研究里程碑。

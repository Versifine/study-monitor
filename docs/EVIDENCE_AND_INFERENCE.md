# 证据、观察与推断

> Version 1 只实现 Evidence 的保存、引用和查询。Observation/Inference 的以下规则约束未来扩展，不要求 M0-M6 建表、排队或运行模型。

## 1. 为什么要严格分层

模型会升级、误判，也可能暂时无法理解复杂手写内容。如果只保留模型结论，未来无法纠正。

## 2. 例子

### Evidence

- 19:00 至 19:20 的书桌视频
- 19:03、19:07、19:12 的高清纸面图
- 同时段电脑窗口事件
- 手机解锁和前台应用事件

### Observation

- 19:04 检测到开始书写
- 19:09 检测到划除两行
- 19:14 检测到打开参考答案页面

### Inference

- 本次作答可能不是完全独立完成
- 当前对“等价无穷小”迁移能力证据不足

## 3. 输出要求

每条推断必须回答：

- 依赖哪些证据
- 有哪些不确定性
- 是否存在替代解释
- 置信度是多少
- 是否适合影响正式学习计划

## 4. 拒绝判断

系统应允许输出：

- 数据不足
- 纸面遮挡
- 题目无法确定
- 过程不连续
- 只能确认最终答案，无法确认思路

拒绝判断优于编造精确分数。

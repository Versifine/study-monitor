# Changelog

## 1.0.1 - M6 认证候选

- 增加密封冻结清单、5 分钟资源采样、每日完整备份/数据库完整性与潜在重复扫描、媒体抽样、核心来源覆盖率门槛和 14 天最终硬拒绝工具。
- `operations/status` 增加只读 Go 协程、堆、栈和 runtime 系统内存计数；不改变 readiness、写入、schema 或保留行为。
- smoke 和故障注入可显式复用已冻结候选二进制，避免用重新构建的不同产物冒充 M6 候选证据。
- ActivityWatch 分页允许同一事件在边界处被截断为不同 duration，并对 data 对象做确定性规范化；仍拒绝同一 id 的真实内容冲突，且不会提交刚好触及迟到截止线的可变事件。
- 增加 M6 执行手册、部署参数模板和最终报告模板；工具完成不提前启动 14 天，也不进入后续智能范围。
